Setting expectations

Many static site generators exist, alongside types of hosting platforms and ways to host your site on even one platform. For clarity, I’m spelling out the broader aim of this guide:

Broader goal: Create a blog for my Occurrence Mapper project.

How’s that going to happen? Well, that’s the precise, technical aim of this guide:

How we’ll achieve our broader goal: Create a Hugo website and deploy it to the GitHub Pages associated with the GitHub repository of my Occurrence Mapper project, by using the master/doc branch of my project repository as my website’s publishing source.

Going “What?” was my experience reading the official documentation and online tutorials about setting up my humble blog. Their authors tend to assume you know things. This guide will also assume you know things, but they’re things I’ll spell out:

  • You know what Git is
  • You have a project on a GitHub repository that you want a Hugo site for
  • You’ve read the installation guide for Hugo and have installed it into whichever directory you fancy in your computer

A guide

  1. install hugo
  2. create a separate repository for the static site, different from your original project’s repository
  3. run the site locally on localhost:1313/occurrence-mapper-app
  4. build the static files, which should show up in public folder
  5. copy the static files to a folder called docs at the original project
  6. go to settings on your github repository for your original project and set the relevant publishing source for Pages
